# agent1 작업 가이드 (최적화)

## 1. 역할

- agent1은 `.gem` 문서를 기준으로 task를 구현하고 결과를 기록한다.
- 상세 코딩 규칙은 `rule1.md`를 단일 기준으로 따른다.

## 2. 필수 문서와 확인 순서

1. `.gem/analylist.md`
2. `.gem/rule1.md`
3. `.gem/task1.md` (또는 사용자 지정 task 문서)
4. `.gem/agent1-worklog.md`

- 작업 시작 시 위 순서를 고정한다.

## 3. 작업 절차

1. task 문서에서 미완료 항목(`- [ ]`)만 추린다.
2. 완료 항목(`- [x]`)은 재실행하지 않는다.
3. 각 task마다 관련 코드 위치를 먼저 확인한다.
4. 수정 범위를 선언한 뒤 구현한다.
5. 구현 완료 후 `gradlew test` 등으로 검증한다.
6. **최종적으로 `.gem/agent1-worklog.md`에 작업 내용을 기록한다.**
7. 선언 범위를 벗어난 수정이 필요하면 사유를 남긴다.

## 4. 완료 기준 (Definition of Done)

1. 코드 반영 완료
2. 변경 코드 주석 반영 완료
3. task 문서 체크박스 반영 완료
4. `gradlew test` 또는 `gradlew compileJava` 1회 성공

## 5. 최종 보고 형식

1. 수정 파일
2. 구현 내용
3. 검증 결과
4. 남은 리스크/미완료 항목

## 6. 작업 이력 기록 규칙

1. 작업 완료 후 `.gem/agent1-worklog.md`에 로그를 추가한다.
2. 로그는 append only로 누적한다.
3. `작업 순서` 번호를 1씩 증가시켜 기록한다.
